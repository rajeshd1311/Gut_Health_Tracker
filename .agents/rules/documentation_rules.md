# Documentation Rules

This rule is active at all times in this workspace.

## Constraints

* **Timestamp Requirement:** Every markdown document, artifact, plan, or brainstorming file generated or updated by the AI assistant in this workspace MUST include a "Last Updated" timestamp or date (e.g., `**Last Updated:** June 7, 2026`) directly under the main title header.

* **Document Categorization Rules:** Document files must always be placed in the appropriate subdirectory based on their purpose:
  * **Brainstorming / Ideation:** Place all brainstorming, ideation sessions, failure modes, and mitigation strategies under `docs/brainstorm/`.
  * **Governance / Core Specifications:** Place all core requirements, specifications, and reviews (such as PRD, Architecture, and Code Reviews) under `docs/governance/`.
  * **Plans / Execution:** Place all implementation plans, roadmaps, and execution specs under `docs/plans/`.
